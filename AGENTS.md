Repository Instructions
=======================

Start of every session
----------------------
1. Read `TECHNICAL_CASES.txt` before doing anything else.
2. Use it as project memory for previously solved tricky technical bugs.

When solving a new tricky issue
-------------------------------
1. Append a new dated entry to `TECHNICAL_CASES.txt`.
2. Record the symptom, root cause, resolution, files touched, and future notes.
3. Keep entries concise but specific enough to help future debugging.

Scope
-----
These instructions apply to work in this repository only.
